memory_leak wrote:
> Mnjah, I doubt MS included strings.h with their runtime. 


No, I don't think so either. But the include in getopt.c is actually including string.h NOT strings.h. If you look at the first line in the code snippet you sent me, there are two conditions.

First: HAVE_STRING_H - Which may or may not be defined depending on your setup (it set on my machine due to the libtiff library).

Secondly: WIN32 - This is defined as true on all windows builds, both x86 and x64. 

Since the conditions are separated with an "or" sign ||. Only one of them have to be true. So I have a really hard time seeing how you would end up trying to include the "strings.h" header.
