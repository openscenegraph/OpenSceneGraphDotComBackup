Ok, it is rarely a good idea to build against master. In the Readme file there are specfied which tagged version which each library is tested against. I have just merged a pull request to support the latest stable version of cUrl (7.49.1).

Also, there is no need to comment things out in the top level cmake file. If you do not want to include a certain library, just leave the corresponding *_SOURCE_DIR variable empty and the build script will omit the library from the build.
