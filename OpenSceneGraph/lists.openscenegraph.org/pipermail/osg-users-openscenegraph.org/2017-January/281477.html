On 28 December 2016 at 23:32, Aaron Andersen <aaron at fosslib.net> wrote:
> Looking at the GLBeginEndAdapter there isn't much documentation so I was

> lacking (which I why enjoy using a rendering engine like OSG :-).

I would not recommend using GLBeginEndAdapter.  It's a transitional
workaround/helper class that served a purpose to help adapt old style
OpenGL code when using GL versions that only support vertex arrays.

The transitional work is now complete on the OSG side so that github
master no longer has any old style OpenGL usage so I have been able to
remove the GLBeginEndAdapter class completely.  This was always the
plan, all future version of the OSG will no have this class.

I would recommend you invest the time in working out how to implement
you code using modern OpenGL vertex arrays, it will result in faster
code - the GLBeginEndAdapter could never be as efficient as you just
creating the data in the correct way in the first place.
