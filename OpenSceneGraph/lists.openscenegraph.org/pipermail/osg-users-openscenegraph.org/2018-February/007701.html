Hi Robert
This idea came from the fact that each time I add a feature I add an associated example...It can become large with time and these examples aren't demos like other examples that use feature in a particular context.
If we accept the typology 
[list=]a test is a unique feature test
a example is a demo (not mature enough to be applications)
an application is a mature (example)[/list]
Then the division makes sense...

And further if we accept this division we can think of a validation application that launch sequentially feature tests to validate osg on particulars users configurations (OS/proc/driver/HW..)
This "validation application" would then be very usefull for user to diagnose non trivial issues ....

But that just my thoughts...
