> However, this crashes my app (my gpu only supports 16.368 pixel

This question has been asked a while ago. Generally the answer boiled 
down to: you most probably can't. OpenGL imposes a limit to the maximum 
texture size. Modern desktop GPUs should handle 8192*8192, but you 
cannot assume to have this. Biggest support I've seen so far was 16384. 
One option is to split the image into pieces and to use multiple 
adjacent quads with the pieces assigned.
The memory issue might be resolved using compressed textures (basically 
using GL_COMPRESSED_RGB for your texture format).

Do you really need to display the complete resolution? If not, the 
simplest option is to resize the texture before displaying it.
