FYI, the Linux WindowSystemInterface implementation uses X11/XRANDR
calls to get the number of displays and resolution etc.  If this is
failing to get the values you expect then I'd guess the window
manager/driver is hiding these settings.  If this is case they you
might need to look at other platform specific API's to get the values.
I'm afraid this is limit of my knowledge on the particular area so
can't say what to use.
