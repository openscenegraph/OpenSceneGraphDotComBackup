Hi Andreas,

On 6/05/2015 13:28, Andreas Schreiber wrote:
> If I pass the following vector from osg code to the shader:
> 
> Code: plane->getOrCreateStateSet()->addUniform(new osg::Uniform("object_position",
> osg::Vec4(1, -0.25, 1,1)));
> 
> This vector is in world space coordinates, correct?

The coordinate system is entirely up to you - tt can be world space, object space, or
anything else.  You are responsible for keeping track of what it is.

But since the uniform is not automagically transformed it could be considered to be in
world space.
