Hi Glen,

In older versions of the OSG there were precision issues on some
databases when doing intersections, this might be happening in your
case. In OSG-3.4 there is support for doing double precision
intersection as well as tightening up of the code to better handle
precisions issues.

Particular problems used to occur when bounding boxes of geometries
were axis aligned so the bounding box was zero thickness.

Robert.
