Hi Dario,

The OSG's binary format is intended to be backwards compatible but
that have been times when this hasn't be possible, sometimes down to
bug fixes, other times perhaps just bugs that are sill lingering,
perhaps the files with animation that you have issues.

I cannot say what these issues might be as I don't have access to the
files that are problematic.  Could you post a link to them?

If Linux is working OK perhaps the best thing to do would be to in
linux convert the files to .osgt and newname.osgb and then try these
resulting files under Windows.

Another possible thing to try is OSG-3.4.1.

Robert
