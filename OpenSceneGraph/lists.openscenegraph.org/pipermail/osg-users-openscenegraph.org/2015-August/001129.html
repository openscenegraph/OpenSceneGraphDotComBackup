Hi John,

Thanks for the error report.  It's curious that I haven't modified
include/osg/Types throughout the rc series and not heard of any problem
reports till yours.  The only change I've made is to include types from
include/osg/GL by adding #include<osg/Types> to the src/osg/GL.in that
CMake uses to generate osg/GL.

When I first introduce the Types header we had a few iteration refining the
_MSC_VER number so had assumed that it was now solid.  This makes me wonder
what might be going on.

A question for everyone, is there a formal listing of the _MSC_VER for the
various VisualStudio versions?

Robert.
